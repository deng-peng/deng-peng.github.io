总结一下python爬虫常用的几个库

  • crawler
  • python

说起来，python是我从大学时就开始学习的一门语言，当时实在很反感c的简陋，觉得大蟒蛇才是未来，在无所事事（其实是逃课）的空闲里，就慢慢学习了这门语言，从图书馆借了很多这类的新书（怀念曾经的新书图书馆）。

后来工作了，略微了解了web开发，又想起了这门课，当时的python书籍还不是那么常见，听说有本python编程金典特别好，但是已经绝版。后来在万能的淘宝找到了影印版，原来的32开本也被复印成了大16开，现在还记得大概花了56大元买下（2006年）。买后花了个把月通读，遗憾的是，随后的若干年内，都没有迈入web开发的大门。

书归正传，前段时间使用python做了几个爬虫项目，包括几个新闻站点的内容抓取，股票信息的抓取，以及污染信息的抓取，使用python的原因，主要是因为在linux上部署方便，当然还有丰富的类库资源，下面总结一下最近用到的一些库：

  1. requests http请求模拟，除了使用方便外，也可以自定义复杂的header等参数
  2. pyquery 仿jQuery的一套dom解析库，实际使用发现实现的已经非常全面。这里备选有beautifulsoup和lxml，根据情况选用。
  3. peewee 虽然最开始，我只是把它当做mysql的helper来用，但是这已经是很好的orm框架了，实际使用感觉功能强大，但是我用不了那么多，图方便时使用。
  4. mongoengine 算是mongodb的orm，同样在抓取时图方便使用，性能上不太清楚。使用pip安装时会有问题，参见mongoengine连接数据库出错

因为涉及到的几个抓取程序都是定时执行，或者一次性执行，并不太考虑持续执行时的性能问题，所以更偏重选择使用简单的库，over。
